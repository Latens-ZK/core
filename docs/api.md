# API Reference

> The Latens backend exposes a FastAPI server that orchestrates Bitcoin UTXO indexing,
> Poseidon-Merkle witness generation, and Starknet calldata construction.
> All proof generation is deterministic: the same inputs always produce the same calldata.

**Base URL:** `http://localhost:8000` (dev) · `http://api.latens.xyz` (prod)

**Interactive docs:** `GET /docs` (Swagger UI) · `GET /redoc` (ReDoc)

---

## Rate Limiting

Rate limiting is keyed on the commitment hash, not the IP address — a semantically stronger bound that prevents any single Bitcoin address from flooding proof generation, while not penalising users behind shared IPs (NAT, VPN, corporate proxies).

```
Proof generation: 10 requests / minute per commitment hash
```

Exceeded requests return `HTTP 429` with a standard slowapi response.

## Endpoints

### `GET /health`

Health check for Docker and load balancer probes.

**Response:**
```json
{ "status": "healthy" }
```

---

### `GET /api/snapshot/latest`

Returns metadata for the most recently indexed Bitcoin snapshot.

**Response:**
```json
{
  "block_height": 800000,
  "merkle_root": "0x04c7...",
  "total_addresses": 8,
  "timestamp": "2024-01-01T00:00:00Z",
  "status": "completed"
}
```

**Error (no snapshot):**
```json
{ "detail": "No snapshot found" }
```
`HTTP 404`

---

### `GET /api/snapshot/current`

Alias for `/api/snapshot/latest`. Provided for frontend compatibility.

---

### `GET /api/snapshot/{height}`

Fetch a snapshot at a specific Bitcoin block height.

**Path params:** `height: int`

**Response:** Same schema as `/latest`

**Error (height not found):**
```json
{ "detail": "Snapshot at height {height} not found" }
```
`HTTP 404`

---

### `GET /api/snapshot/status/{height}`

Check whether a snapshot at `height` is available and what its processing status is.

**Response:**
```json
{
  "height": 800000,
  "status": "completed",   // "pending" | "processing" | "completed" | "failed"
  "addresses_indexed": 8
}
```

---

### `POST /api/snapshot/generate`

Trigger generation of a new snapshot at the given block height. Fetches UTXO data from Blockstream API, builds Merkle tree, and persists to the database.

**Request:**
```json
{ "block_height": 800000 }
```

**Response:**
```json
{
  "block_height": 800000,
  "addresses_indexed": 8,
  "merkle_root": "0x04c7...",
  "status": "completed"
}
```

**Notes:**
- For the demo database this completes synchronously (8 addresses)
- For real Bitcoin UTXO set (~50M addresses filtered to ~1M with balance > 0.01 BTC), this runs as a background task
- Same block height called twice is idempotent — returns the cached snapshot

---

### `POST /api/proof/generate`

Core endpoint. Validates the address exists in the snapshot, verifies Merkle inclusion and threshold off-chain, and returns Starknet calldata.

**Request:**
```json
{
  "address": "1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa",
  "salt_hex": "deadbeefdeadbeef...deadbeef",   // 64-char hex (32 bytes)
  "threshold": 100000000                        // satoshis; 0 = no threshold
}
```

**Field constraints:**
- `address`: valid Base58 (P2PKH/P2SH) or Bech32 (native SegWit) Bitcoin address
- `salt_hex`: exactly 64 hex characters — 32 bytes — generated by `crypto.getRandomValues` in browser
- `threshold`: non-negative integer in satoshis; `100000000 = 1 BTC`

**Success response:**
```json
{
  "proof": "LATENS_PROOF_v1:commitment=0x...:root=0x...:depth=3:threshold=100000000",
  "public_signals": {
    "snapshot_root": "0x04c7...",
    "commitment": "0x1a2b...",
    "threshold": 100000000
  },
  "block_height": 800000,
  "starknet_calldata": [
    "0x1a2b...",   // address_hash
    "0xdead...",   // salt
    "5000000000",  // balance (satoshis as felt252)
    "3",           // merkle_path length
    "0xabc...",    // path[0].value
    "1",           // path[0].direction
    "0xdef...",    // path[1].value
    "0",           // path[1].direction
    "0x789...",    // path[2].value
    "1",           // path[2].direction
    "0x1a2b...",   // commitment
    "100000000"    // threshold
  ],
  "verified_locally": true,
  "calldata_ready": true
}
```

`starknet_calldata` is the exact array to pass to `BalanceVerifier.verify_proof()`. The frontend maps this directly to the contract call with no additional transformation.

**Error responses:**

| Error code | HTTP | Trigger |
|---|---|---|
| `address_not_indexed` | 404 | Address not found in current snapshot |
| `threshold_not_met` | 400 | `balance < threshold` |
| `invalid_address` | 422 | Malformed Bitcoin address format |
| `snapshot_unavailable` | 503 | No snapshot in database yet |

Error body:
```json
{
  "error": "address_not_indexed",
  "message": "Address not included in current snapshot. Minimum balance not met or address has no UTXOs at block 800000."
}
```

### `GET /api/snapshot/witness/{address}`

Returns the raw Merkle witness (path + balance) for a given address. Does not perform circuit verification or construct calldata. Useful for client-side proof generation implementations.

**Response:**
```json
{
  "address_hash": "0x1a2b...",
  "balance": 5000000000,
  "merkle_path": [
    { "value": "0xabc...", "direction": true },
    { "value": "0xdef...", "direction": false },
    { "value": "0x789...", "direction": true }
  ],
  "snapshot_root": "0x04c7...",
  "block_height": 800000
}
```

**Use case:** S2/S3 client-side proving. The browser fetches the witness, runs the WASM prover locally, and never sends the salt or address hash to the backend.

## Internals: Proof Generation Pipeline

```
POST /api/proof/generate
    │
    ├─ Validate BTC address format (base58/bech32 checksum)
    │
    ├─ Compute address_hash = address_bytes_as_int % P
    │
    ├─ DB lookup: AddressBalance where address_hash matches
    │  └─ If missing → address_not_indexed
    │
    ├─ Compute commitment = Poseidon(address_hash, salt)
    │
    ├─ Retrieve merkle_path for address_hash from snapshot
    │
    ├─ ProofGenerator.generate_proof_no_salt(...)
    │  ├─ Assert balance >= threshold        → threshold_not_met
    │  └─ Assert recompute_root == DB_root   → merkle_mismatch (bug)
    │
    ├─ ProofGenerator.generate_calldata(...)
    │
    └─ Return { proof, public_signals, starknet_calldata, ... }
```
